## Hard-won insights

### SubagentStop vs Stop — critical distinction
The original Stop hook fired on every session end, regardless of whether implement-ticket ran. It read the most-recent session JSONL without filtering, producing counts from unrelated sessions. The fix was `SubagentStop` + `meta.json agentType` guard. **Non-obvious:** the subagent must have a `meta.json` alongside its JSONL for the hook to detect it — if the file is absent, the hook silently skips.

### TRUNCATE destroys Flyway seed data
`TRUNCATE ... RESTART IDENTITY CASCADE` resets sequences and cascades to all rows. In a DB where test data lives at `partner_number < 1,000,000` and bulk data at `>= 1,000,000`, a TRUNCATE followed by bulk reload produces a dataset that passes load checks but silently fails e2e tests that look up seed partners by number.

### ES container rename propagation
When a Compose service is renamed (e.g., `elasticsearch-spring-partner` → `elasticsearch-partner` in PARTNER-ES-018), shell scripts that reference the old name silently succeed — `docker compose stop <wrong-name>` exits 0 even when the service doesn't exist. Grep all `<domain>/scripts/` for old names after any Compose service rename.

### PARTNER-FW-007 token cost
5.5M cache-read tokens for a frontend-only ticket. The concurrent-async race condition fix (`discarded` flag, generation counter) required frontier reasoning. The streaming `combineLatest` pattern itself was straightforward once the design was clear. Future similar tickets (adding a fifth backend permutation) would be local-LLM-safe given the documented pattern.
