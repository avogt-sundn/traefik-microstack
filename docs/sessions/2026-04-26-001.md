## Hard-won insights

The `e2e-tests` Docker image must be rebuilt explicitly after editing files under `tests/`. The `build-if-changed.sh` script does not map the `e2e-tests` service to a domain (the service is absent from `service-domain.sh`), so `make rebuild SERVICE=e2e-tests` exits with error code 1. Workaround: `docker compose build e2e-tests` directly.

The browser Playwright project (`PW_PROJECT=browser`) is separate from the API project (`PW_PROJECT=api`). `make pw-run` only runs the API project. Browser tests must be triggered separately via `docker run ... -e PW_PROJECT=browser`.
