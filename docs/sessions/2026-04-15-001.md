## Hard-won insights

**The signal for "should this be an agent?":** Count the subagents it spawns. If the answer is > 1, it is an orchestrator — and orchestrators belong in `.claude/agents/`, not `.claude/skills/`. Skills are single-pass; agents can have phases, branches, and delegation chains.

**Stub backward compatibility:** The skill stub (`SKILL.md`) keeps the `/implement-ticket` invocation path working without requiring callers to change. This is the correct migration pattern — leave a forwarding stub at the old location.

**Agent routing advantage:** Agent frontmatter (`subagent_type`, `model`, `color`) is metadata that skills lack. Other agents can now reference `implement-ticket` by type in their own delegation instructions.
