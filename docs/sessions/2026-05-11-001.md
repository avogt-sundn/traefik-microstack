## Hard-won insights

- WebLLM (MLC/WebGPU) can carry a larger model than Transformers.js (ONNX) at the same hardware tier: desktop WebLLM uses 7B, Transformers.js caps at 3B. Reason: MLC quantization is more memory-efficient than ONNX execution.
- `screen.width < 768` + iOS/Android regex is a pragmatic tablet-safe threshold: tablets report "Android" but have >768px width, so they route to the desktop tier. Accurate enough for a lab demo.
- The three sanctioned extension points in PGE (federation.manifest.json, @traefik-microstack/shared, docker-compose include:) map exactly to the three integration seams already present in the monorepo. PGE is not a new process — it names what is already in place.
- The UX Schichtenarchitektur (foundations → patterns → screens → prototype) is a direct application of CLAUDE-16 ordered siblings to the UX layer. Writing this down clarified that the axiom already implied the UX architecture.
