## Key decisions

- [ADR-0013](../decisions/ADR-0013-bounded-context-no-cross-domain-db-reads.md): No cross-domain DB reads — `partner-edit` pushes full payload to `partner-search` index endpoint.
- `PartnerEditDto` is a hand-crafted domain DTO (13 fields), not a generated superset. OpenAPI generation was kept for the server side only.
- Tab reduction: only Address and Gruppe tabs remain in the partner-edit form. The other 6 tabs were empty stubs.

## Hard-won insights

**PartnerDto shape mismatch was doubly silent.** GET loaded nothing because the template bound to `contactPersons[0].name1` (OpenAPI nested shape) while the backend returned flat fields. PUT sent nulls because Angular serialized the nested DTO shape while `EditRequest` expected flat fields. Neither failure threw an exception — both manifested as blank UI / silent no-op save.

**`make up` auto-rebuilds via `build-if-changed.sh`.** The Makefile `build-changed` target calls `scripts/build-if-changed.sh` per service. Images are rebuilt when source changes. Inserting `make rebuild` between code edits and `make up` in the implement-ticket flow is unnecessary and adds ~2 minutes per cycle.

**Disk exhaustion.** `make up` failed with "No space left on device" — overlay at 100%. Fixed by `docker volume prune -f` (~35 GB freed). Stack came up cleanly. Do NOT use `docker system prune -af` — it destroys locally-built images and forces a cold rebuild.
