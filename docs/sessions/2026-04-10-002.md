## Key decisions

- [ADR-0005](../decisions/ADR-0005-angular-isdevmode-build-tool-distinction.md): `isDevMode()` is unfit in Native Federation; esbuild `define` is not passed through by the native-federation builder; `fileReplacements` is the correct solution

## Hard-won insights

### Root cause 1 (immediate): stale container

The `make pw-local` Playwright tests failed with a timeout waiting for `[cypressid="cypress-partner-search-input"]` — the partner MFE was not rendering. The container had been rebuilt mid-session while `app.ts` referenced `IS_DEV_BUILD` (a global constant) but `angular.json`'s `define` option had not yet been applied. At runtime `IS_DEV_BUILD was not defined`, crashing the Angular bootstrap. Fix: revert `app.ts` to committed state, rebuild container cleanly.

### Root cause 2 (deeper): `isDevMode()` always returns `false` through the shell

In Native Federation, `@angular/core` is a shared singleton. The production shell bootstraps first and registers its production-compiled `@angular/core` with `ngDevMode = false`. When the partner remote loads, the federation runtime returns the shell's already-registered module — it never initialises its own copy. Every call to `isDevMode()` from the partner goes through the shell's production module: always `false`.

This is not a build configuration error — it is a fundamental property of module federation's shared singletons. No amount of configuring the partner's own build fixes it.

### Root cause 3: esbuild `define` not passed through

Replacing `isDevMode()` with a global constant and injecting it via `define` in `angular.json` was the first attempted solution. It did not work: the content hash of the component chunk was unchanged after adding `define`, confirming esbuild never saw the option. The `@angular-architects/native-federation:build` builder does not forward `define` to the underlying `@angular/build:application` invocation.

### Correct solution: `fileReplacements`

`fileReplacements` is a first-class Angular build feature supported through the full builder chain. The production file exports `IS_DEV_BUILD = false`; the development replacement exports `IS_DEV_BUILD = true`. The constant is resolved at compile time, lives entirely in the partner bundle, and is never sourced from the shared `@angular/core`.
