## Goal

Scaffold the `ux/frontend/` Storybook 8 project so the UX domain has a live, buildable pattern library served through Traefik at `/ux`.

## Context

The `ux/` domain was created to own the design system for the whole application. Before any patterns can be documented as live stories, the Storybook project, Dockerfile, and Compose service must exist. Nothing previously existed under `ux/frontend/`.

## Acceptance criteria

- [x] `ux/frontend/package.json` defines `storybook dev` and `build-storybook` scripts using Storybook 8 + `@storybook/html-vite`
- [x] `ux/frontend/.storybook/main.ts` configures the `html-vite` framework and `addon-essentials`
- [x] `ux/frontend/stories/DetailForm.stories.ts` has `ReadOnly` and `Editable` story variants
- [x] `ux/frontend/Dockerfile` builds via `node:24` → serves static output via `nginx:stable` on port 80
- [x] `ux/docker-compose.yaml` defines `ux` (priority 120, strip `/ux`) and `forward-ng-ux` (priority 1100, port 6006)
- [x] `docker-compose.yaml` root includes `ux/docker-compose.yaml`
- [x] `docker compose build ux` succeeds and produces a working image
- [x] `docker compose up -d ux` results in a healthy container; Traefik registers `ux@docker` as `enabled`
- [x] `npx storybook dev --port 6006` starts the dev server; `forward-ng-ux` becomes healthy and Traefik routes `/ux` to it at priority 1100
